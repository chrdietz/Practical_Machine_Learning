Practical Machine Learning - Quantified Self Data Analysis

Task Background

Using devices such as Jawbone Up, Nike FuelBand, and Fitbit it is now possible to collect a large amount of data about personal activity relatively inexpensively. These type of devices are part of the quantified self movement a group of enthusiasts who take measurements about themselves regularly to improve their health, to find patterns in their behavior, or because they are tech geeks. One thing that people regularly do is quantify how much of a particular activity they do, but they rarely quantify how well they do it. In this project, your goal will be to use data from accelerometers on the belt, forearm, arm, and dumbell of 6 participants. They were asked to perform barbell lifts correctly and incorrectly in 5 different ways. More information is available from the website here: http://groupware.les.inf.puc-rio.br/har (see the section on the Weight Lifting Exercise Dataset).

The goal of your project is to predict the manner in which they did the exercise. This is the classe variable in the training set. You may use any of the other variables to predict with. You should create a report describing how you built your model, how you used cross validation, what you think the expected out of sample error is, and why you made the choices you did. You will also use your prediction model to predict 20 different test cases.

Download and read the data

In the first step we download the raw training and testing data sets and directly assign them to the respective variables.

# Prepare download of the data.

Cleaning the data

Now that we created two separate variables that contain the raw data for training and testing, the ext step is to remove all columns that contain NA values

trainRawData <- trainRawData[, colSums(is.na(trainRawData)) == 0] 
testRawData <- testRawData[, colSums(is.na(testRawData)) == 0]

Removing some variables

#Extracting the class label
classe <- trainRawData$classe

#Removing some variables in the training set
toRemoveInTrain <- grepl("^X|timestamp|window", names(trainRawData))

trainRawData <- trainRawData[, !toRemoveInTrain]
cleanTrainingSet <- trainRawData[, sapply(trainRawData, is.numeric)]
cleanTrainingSet$classe <- classe

#Ensuring to remove the same variables as removed in the training set
toRemoveInTest <- grepl("^X|timestamp|window", names(testRawData))
testRawData <- testRawData[, !toRemoveInTest]
cleanTestSet <- testRawData[, sapply(testRawData, is.numeric)]

Slicing the data

We use a fixed seed in order to make our modelling process reproducible. Then we extract 70% of the cleaned training data set for the actual training and remaining 30% for the validation.

set.seed(12345)

myTestData <- cleanTrainingSet[-inTrainingSet, ]

Modelling

To fit the predictive model for activity recognition, we use the Random Forest algorithm. Using this algorithm is beneficial, because it automatically ranks and selects important variables. In addition, it is quite robust to outliers. We use a 10-fold cross-validation for applying our algorithm. We set the number of trees to be included in our model to 200.

tC <- trainControl(method="cv", 10)
rf <- train(classe ~ ., data=myTrainData, method="rf", trControl=tC, ntree=200)
